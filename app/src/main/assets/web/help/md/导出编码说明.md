# 导出编码说明

## 概述

在导出书籍为 TXT 格式时,需要选择字符编码。字符编码决定了文本如何转换为计算机可存储的字节序列。选择正确的编码非常重要,否则可能导致导出的文件在其他设备或软件中打开时出现乱码。

## 支持的编码列表

阅读Max 支持以下导出编码:

- **UTF-8** (推荐)
- **GB2312**
- **GB18030**
- **GBK**
- **Unicode**
- **UTF-16**
- **UTF-16LE**
- **ASCII**

## 编码详解

### UTF-8 (强烈推荐)

**特点:**
- 国际标准编码,兼容性最好
- 支持所有语言字符(中文、英文、日文、韩文等)
- 变长编码,英文占 1 字节,中文占 3 字节
- 网络传输和跨平台交换的首选编码
- 绝大多数现代软件和设备都完美支持

**适用场景:**
- 导出后在电脑、手机、平板等多种设备阅读
- 分享给他人阅读(对方可能使用不同设备)
- 上传到云端或网络平台
- 需要长期保存的书籍文件

**小白推荐指数:** ⭐⭐⭐⭐⭐ (5/5)

---

### GB2312

**特点:**
- GB2312 是中国大陆第一个正式的国家级汉字编码标准
- 中国国家标准简体中文字符集
- 包含 6763 个汉字和 682 个非汉字图形字符
- 每个字符固定占 2 字节
- 主要支持简体中文和 ASCII 字符
- 不支持繁体中文、日文、韩文等

**适用场景:**
- 只包含简体中文和英文的书籍
- 需要在老旧设备或软件上打开(如某些旧版电子书阅读器)
- **在MP4播放器、电子词典、老式电子书设备上阅读** ⭐
- 文件体积相对较小(中文部分比 UTF-8 少 1 字节)

**特别说明:**
很多老设备和特殊设备**只支持 GB2312 编码**,包括:
- 📱 **MP4播放器** - 大多数国产MP4只识别GB2312编码的TXT文件
- 📖 **老式电子书阅读器** - 早期电子书设备(如某些国产阅读器)
- 📚 **电子词典** - 学习机、电子词典等教育设备
- 🖥️ **老旧手机** - 早期非智能手机、功能机
- 📟 **车载MP3/MP4** - 某些车载播放设备

这些设备通常固件简单,只内置了GB2312解码器,无法识别UTF-8编码的文件。如果你需要在这些设备上阅读导出的书籍,**必须使用GB2312编码**。

**注意事项:**
- 如果书籍包含繁体字、特殊符号或非中英文字符,会导致乱码或丢失
- 确保书籍内容只有简体中文和英文,否则会出错

**小白推荐指数:** ⭐⭐⭐⭐ (4/5) - 特殊设备场景

---

### GBK

**特点:**
- GB2312 的扩展版本
- 支持简体中文、繁体中文、日文假名等
- 包含 20902 个汉字
- 每个字符固定占 2 字节
- Windows 系统中文环境常用编码

**适用场景:**
- 包含繁简中文的书籍
- 主要在 Windows 系统上阅读
- 与旧版 Windows 软件兼容性较好

**注意事项:**
- 跨平台兼容性不如 UTF-8
- 部分非 Windows 设备可能不支持

**小白推荐指数:** ⭐⭐⭐⭐ (4/5)

---

### GB18030

**特点:**
- 中国最新国家标准编码
- GBK 的超集,包含所有 GBK 字符
- 支持所有 Unicode 字符(通过四字节编码)
- 与 UTF-8 一样可以表示任何字符
- 变长编码(1/2/4 字节)
- 中国政府强制标准编码

**适用场景:**
- 需要符合国家标准的应用场景
- 包含少数民族文字、特殊符号的书籍
- 在中国境内的软件系统中使用

**注意事项:**
- 国际兼容性不如 UTF-8
- 部分老旧软件可能不支持四字节编码部分

**小白推荐指数:** ⭐⭐⭐⭐ (4/5)

---

### Unicode (UTF-16)

**特点:**
- Unicode 标准编码之一
- 每个字符占 2 或 4 字节
- 支持所有语言字符
- Windows 内部常用编码

**适用场景:**
- Windows 系统内部处理
- 特定编程环境或开发需求

**注意事项:**
- 文件体积较大(英文也占 2 字节)
- 跨平台兼容性较差
- 很多移动设备不支持
- 不适合普通用户使用

**小白推荐指数:** ⭐ (1/5)

---

### UTF-16LE

**特点:**
- UTF-16 的小端序版本
- Windows 系统内部默认编码
- 每个字符占 2 或 4 字节

**适用场景:**
- Windows 系统特定应用
- 开发调试用途

**注意事项:**
- 文件体积大
- 跨平台兼容性差
- 普通用户很少需要

**小白推荐指数:** ⭐ (1/5)

---

### ASCII

**特点:**
- 最基础的编码
- 只支持 128 个字符(英文、数字、基础符号)
- 每个字符固定占 1 字节
- 不支持任何中文或其他语言

**适用场景:**
- 纯英文书籍
- 特殊技术用途

**注意事项:**
- **无法编码任何中文字符**
- 如果书籍包含中文会导致严重错误

**小白推荐指数:** ❌ (不推荐中文用户)

---

## 小白用户选择指南

### 推荐选择

**UTF-8** 是小白用户的最佳选择:

✅ **兼容性最好** - 所有设备、软件都支持
✅ **通用性强** - 适合所有语言和字符
✅ **无乱码风险** - 不会出现字符丢失或显示错误
✅ **网络友好** - 上传云端、分享他人都无问题
✅ **长期保存** - 未来几十年都不会过时

### 其他选择场景

| 场景 | 推荐编码 | 原因 |
|------|---------|------|
| 日常导出 | UTF-8 | 兼容性最好,无后顾之忧 |
| 分享给他人 | UTF-8 | 对方设备兼容性不确定 |
| 上传云端 | UTF-8 | 云平台标准编码 |
| 只在 Windows 电脑阅读 | GBK 或 UTF-8 | GBK 体积稍小,UTF-8 更通用 |
| 纯简体中文书籍 | GB2312 或 UTF-8 | GB2312 体积小,UTF-8 更安全 |
| 包含繁简中文 | GBK 或 UTF-8 | GBK 支持繁体,UTF-8 最安全 |
| 纯英文书籍 | UTF-8 或 ASCII | UTF-8 更通用,ASCII 体积最小 |

### 不推荐的选择

❌ **Unicode/UTF-16/UTF-16LE** - 文件体积大,兼容性差
❌ **ASCII** - 无法编码中文,会导致严重错误
❌ **GB2312** - 如果书籍包含繁体字或特殊符号会出错

## 如何设置导出编码

### 方法一:在缓存与导出界面设置

1. 打开「缓存与导出」界面
2. 点击右上角菜单按钮(三个点)
3. 选择「导出编码」
4. 在弹出的对话框中输入编码名称(如 `UTF-8`)
5. 点击确定保存

### 方法二:在导出对话框设置

部分导出场景会在导出前弹出设置对话框,可以直接选择编码。

## 常见问题

### Q: 为什么导出的文件在其他设备打开是乱码?

**A:** 可能原因:
1. 编码选择不当(如用 GB2312 编码包含繁体字的书籍)
2. 打开文件的软件不支持该编码
3. 设备系统默认编码与文件编码不匹配

**解决方案:** 使用 UTF-8 编码导出

### Q: UTF-8 文件体积会不会太大?

**A:** 相比 GB2312/GBK:
- 英文部分:体积相同(都是 1 字节)
- 中文部分:UTF-8 多 1 字节(3 字节 vs 2 字节)

对于一本 10 万字的中文小说:
- GB2312: 约 200KB
- UTF-8: 约 300KB

差异约 100KB,对于现代设备存储空间来说微不足道。换来的是完美的兼容性,非常值得。

### Q: 我应该长期使用 GBK 吗?

**A:** 不建议。虽然 GBK 在 Windows 系统上兼容性好,但:
- 跨平台兼容性不如 UTF-8
- 未来可能遇到不支持 GBK 的新设备
- UTF-8 是国际标准,长期趋势

除非有特殊需求(如必须在老旧设备上阅读),否则建议使用 UTF-8。

### Q: 如何查看当前导出编码?

**A:** 在「缓存与导出」界面菜单中,「导出编码」菜单项会显示当前编码,如「导出编码(UTF-8)」。

### Q: 导出编码和阅读编码有什么区别?

**A:**
- **导出编码**:导出书籍文件时使用的编码,决定文件内容如何保存
- **阅读编码**:打开本地 TXT 文件时识别编码的方式,阅读Max 会自动检测

导出编码需要手动设置,阅读编码通常自动识别。

## 总结

### 通用场景:UTF-8 是最佳选择

对于大多数小白用户,**UTF-8 是唯一需要记住的编码**:

- ✅ 无脑选择 UTF-8,永远不会出错
- ✅ 兼容所有现代设备和软件
- ✅ 支持所有语言字符
- ✅ 适合长期保存和分享

除非有明确的技术需求或特殊兼容性要求,否则**请始终使用 UTF-8 编码导出书籍**。

### 特殊设备场景:GB2312 是必须选择

如果你使用以下老设备,**必须使用 GB2312 编码**:

- 📱 MP4播放器/MP5播放器
- 📚 电子词典、学习机
- 📖 老式电子书阅读器
- 🖥️ 老旧功能手机
- 📟 车载播放设备

这些设备固件简单,只内置了GB2312解码器,**无法识别UTF-8编码的文件**。使用UTF-8会导致乱码或无法打开。

**使用GB2312的前提条件:**
- 书籍内容**只有简体中文和英文**
- 不能包含繁体字、特殊符号、日韩文等
- 否则会导致字符丢失或乱码

### 编码选择决策流程

```
需要导出书籍 → 判断使用场景
    ↓
是否在MP4/电子词典/老设备上阅读?
    ↓
是 → 使用 GB2312 (确保内容只有简体中文+英文)
    ↓
否 → 使用 UTF-8 (通用场景,无后顾之忧)
```
